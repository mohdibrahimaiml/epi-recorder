Enterprise

Org seals the run.
Humans add bound reviews.
Auditors verify offline.

EPI is a file-first evidence layer — not a dashboard login. Cryptography proves integrity and sealer keys; free-text names are claims, not HR identity.

pip install epi-recorder Verify a file

Acceptance gate

Commands security and CI should run.

epi keys generate --name org-seal
epi keys bundle-export --out org-trust-bundle.zip
# seal artifacts in CI with that key

# Auditor / CI:
epi keys bundle-import org-trust-bundle.zip
epi verify artifact.epi --policy strict
epi verify artifact_reviewed.epi --review

Modes

  • Air-gapped seal: EPI_NOTARIZE=0
  • Standard seal: RFC 3161 on by default (network at seal only)
  • Verify / view: integrity + signature offline; identity from local trust store

What PASS means

  • --policy strict → integrity + trusted sealer + completeness
  • --review Binding True → v1.1 human review bound to sealed evidence
  • Standard-policy WARN with unknown key is expected until trust import
