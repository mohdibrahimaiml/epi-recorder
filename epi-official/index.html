Trust Architecture

Three cryptographic primitives. One container.

EPI combines three well-established cryptosystems into a single verifiable artifact — signed, chained, and self-contained.

🔐

Ed25519 Signing

Every .epi file contains a manifest listing each file and its SHA-256 hash. The manifest is signed with Ed25519. RFC 8032 compliant. One byte changes — the signature fails.

⛓

Hash-Chained Timeline

Each execution step carries the SHA-256 hash of its predecessor. Insert, delete, or reorder any step — and the chain breaks. Tamper-evident by construction, not by policy.

📦

Polyglot Container

A .epi file is valid HTML and a binary archive simultaneously. Double-click to open the embedded forensic viewer in any browser. No installation. Air-gap ready.

Trust Model

Who signed it. What it proves.

Every .epi carries a computed trust level based on cryptographic evidence — from HIGH (registry-verified signer) to TAMPERED (integrity failure).

HIGHPassValidKnownTrust registry verified signer identity
