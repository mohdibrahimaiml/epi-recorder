ARCHITECTURE

THE ARCHITECTURE
OF TRUST

Technical deep dive into the cryptographic trust system. For engineers implementing EPI and regulators evaluating its evidentiary properties.

// 1.0 CONTAINER FORMAT

EPI uses a dual-format container. Envelope v2 is a polyglot HTML+ZIP format (128-byte binary header + embedded viewer + ZIP payload) that opens in any browser while remaining cryptographically verifiable. Legacy ZIP is fully supported for backward compatibility.

ENVELOPE V2 (CURRENT)

128-byte binary header with magic bytes, version, UUID, payload SHA-256. Polyglot enables zero-install browser viewing.

LEGACY ZIP

Standard ZIP with mimetype as first entry. Auto-detected on read. Supported for all earlier artifacts.

// 2.0 CRYPTOGRAPHIC TRUST MODEL

SHA-256 INTEGRITY

Every file hash-checked against manifest.json. Extra files flagged as injections. Missing files reported.

ED25519 SIGNATURES

RFC 8032 Ed25519 with canonical JSON hashing (RFC 8785 style). Format: ed25519:keyname:hexsig.

PREV-HASH CHAIN

Each step links to predecessor. Inserting/removing/reordering breaks the chain — detected by verify.

IDENTITY RESOLUTION

TrustRegistry: local keys, DID:WEB, remote registry. Without registry, identity is explicitly UNKNOWN.
