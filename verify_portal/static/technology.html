Architecture

The architecture of trust

Technical deep dive into the cryptographic system behind EPI. For engineers and regulators.

1. Container Format

Polyglot HTML+ZIP. 128-byte binary header with magic bytes, version, UUID, payload SHA-256. Opens in any browser. Legacy ZIP format fully supported.

Envelope v2

128-byte binary header. Magic bytes, version, UUID, payload SHA-256. Polyglot for zero-install browser viewing.

Legacy ZIP

Standard ZIP with mimetype first. Auto-detected on read. Backward compatible with all earlier artifacts.

2. Cryptographic Trust Model

SHA-256 Integrity

Every file hash-checked against manifest. Missing/extra files detected. Mismatches fail verification.

Ed25519 Signatures

RFC 8032. Canonical JSON hashing. Format: ed25519:keyname:hexsig. 32-byte public keys.

Prev-Hash Chain

Each step links to predecessor via SHA-256. Insert, remove, reorder — detected by verify.

Identity Resolution

TrustRegistry: local keys, DID:WEB, remote registry. Unknown = explicitly marked.

3. Fault Analysis (9 Passes)
Pass 1

Error Continuation — agent proceeds after error without review

Pass 2

Constraint Violation — input/output exceeds defined bounds

Pass 3*

Sequence Violation — steps out of policy-defined order

Pass 4*

Threshold Violation — numeric limits exceeded

Pass 5*

Prohibition Violation — forbidden action attempted

Pass 6

Approval Gap — human review required but missing

Pass 7*

Approval Guard — threshold not met

Pass 8

Context Drop — critical info omitted mid-execution

Pass 9*

Tool Permission — tool invoked outside allowed set

4. SCITT & Redaction

SCITT Transparency

COSE Sign1 via cbor2 + cryptography. Anchoring requires deployed SCITT service.

Secret Redaction

API keys, tokens, PII scanned pre-seal. HMAC-SHA256 placeholders. Secrets never touch evidence.
