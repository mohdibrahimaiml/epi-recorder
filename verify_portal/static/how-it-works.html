How EPI Works

Three phases. One portable file. Verifiable forever.

1 Record

Wrap your AI agent with EPI's context manager. Every LLM call, tool invocation, approval decision, and error is captured automatically into a portable .epi file.

from epi_recorder import record

with record("agent-run.epi", goal="Loan decision pipeline"):
    response = client.chat.completions.create(...)

# agent-run.epi written — signed and sealed

What's captured: Every API call (request + response + tokens), every tool invocation, every approval, environmental context, timing metadata. Integrated with OpenAI, Anthropic, LangChain, LangGraph, LiteLLM, Guardrails, pytest, and OpenTelemetry.

2 Seal

EPI computes a cryptographic hash chain across every step, signs the manifest with Ed25519, and optionally anchors it to a SCITT transparency service. Once sealed, any modification is detectable.

$ epi verify agent-run.epi --verbose

  [1/5] Structure  PASS
  [2/5] Manifest   PASS  (SHA-256: d4e2f1a8b3c7...)
  [3/5] Integrity  PASS  (all files match)
  [4/5] Hash Chain PASS  (42 steps intact)
  [5/5] Signature  PASS  (Ed25519 verified)

  DECISION: PASS

What sealing proves: The file hasn't been modified. Signatures trace to known keys. Hash chain is intact. SCITT receipt provides third-party transparency.

3 Verify

Verification is completely offline. No server. No API key. No calling home. A .epi file proves itself — email it to an auditor, archive it for a decade, or open it on an air-gapped machine.

Verification methods:

  • Browser: Drop on epilabs.org/verify — Web Crypto + JSZip, entirely client-side
  • CLI: epi verify file.epi — same checks, terminal output
  • CI/CD: uses: epi-recorder/.github/actions/verify-epi@main — automatic in pipelines
