Pipeline

How EPI works

From raw AI execution to a cryptographically sealed evidence file — in five steps that anyone can independently verify.

00

Install

One command. Zero dependencies. macOS, Linux, Windows.

pip install epi-recorder
01

Record

Wrap your client. Open a context. Every call, tool, and decision captured as structured JSONL.

# Three lines of Python
with record("agent.epi"): run()
02

Seal

SHA-256 manifest computed. Ed25519 signature applied. Chain locked. The file is cryptographically sealed.

# Auto-seals on context exit
✓ Sealed · Ed25519 · SHA-256
03

Verify

Five checks: structure, integrity, signature, chain, SCITT. CLI or browser. Nothing uploaded.

epi verify agent.epi
VERIFIED ✓ · HIGH trust
04

Share

Send the .epi file to an auditor. They open it in any browser. Embedded viewer. Offline. Air-gap ready.

# Drop at epilabs.org/verify
Or double-click to open
Anatomy

Inside a .epi file

agent_run.epi

├── environment.json · Full runtime snapshot
├── analysis.json · 9-pass fault analysis
├── viewer.html · Embedded forensic viewer
└── VERIFY.txt · Offline verification guide
